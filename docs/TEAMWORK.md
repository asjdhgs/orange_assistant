# 模块职责说明

项目按照功能边界拆分为多个 Rust 模块，各模块通过统一的应用状态、请求响应结构和错误类型协作。

| 模块 | 主要职责 | 对应文件 | Rust 技术点 |
|---|---|---|---|
| 服务入口与配置 | 加载环境变量、初始化数据库连接、读取知识图谱数据并启动 Web 服务 | `main.rs`、`config.rs` | async/await、Result、Arc、工程组织 |
| 路由与接口 | 组织页面入口、静态资源、认证、推荐、MBTI、知识图谱和模型问答接口 | `routes.rs`、`frontend.rs` | Axum Router、Extractor、SSE、共享状态 |
| 用户认证 | 手机号注册、密码登录、管理员用户查询和内存降级数据 | `auth.rs` | struct、参数绑定、错误传播、并发读写 |
| MBTI 探索 | 题目加载、类型计分、职业方向匹配和结果格式化 | `mbti.rs` | enum、HashMap、迭代器、单元测试 |
| 志愿推荐 | 根据分数、位次、选科、城市和专业偏好筛选院校并计算冲稳保结果 | `recommendation.rs` | 所有权、借用、排序、SQLx 查询 |
| 组合分析 | 计算志愿组合比例、风险等级、分数敏感性和模拟结果 | `portfolio.rs` | 数值计算、随机模拟、纯函数测试 |
| 知识图谱 | 解析专业关系数据，按用户偏好生成相关节点和边 | `knowledge.rs` | HashMap、HashSet、VecDeque、图遍历 |
| 模型问答 | 调用 DeepSeek 流式接口，并在模型不可用时生成本地解释 | `llm.rs` | 异步网络、流式响应、错误降级 |
| 数据结构 | 定义请求、响应、学生画像、推荐结果和图谱节点 | `models.rs` | Serialize、Deserialize、强类型建模 |
| 错误处理 | 将数据库、网络、配置和业务错误转换为统一 HTTP 响应 | `error.rs` | 自定义错误、From 转换、IntoResponse |

## 模块协作流程

1. 用户在前端填写分数、位次、选科、城市和专业偏好。
2. 路由层将请求解析为强类型结构，并交给推荐模块处理。
3. 推荐模块读取 MySQL 数据或内置演示数据，生成院校和专业候选结果。
4. 组合分析模块补充风险、比例和敏感性说明。
5. 模型问答模块根据推荐结果生成自然语言解释。
6. 前端展示推荐表、AI 解读、MBTI 方向和知识图谱。
