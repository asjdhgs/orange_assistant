# 小组成员分工说明

本项目按 3～5 人组队要求整理分工。提交前请补全真实姓名、学号，并保证 Git 提交记录与分工一致。

| 成员 | 建议负责模块 | 主要文件 | 可展示的 Rust 知识点 |
|---|---|---|---|
| 待填写（组长） | API 架构、配置、错误处理、发布集成 | `main.rs`、`routes.rs`、`config.rs`、`error.rs` | async/await、Result、共享状态、工程组织 |
| 待填写 | 用户认证、MBTI 测评与职业推荐 | `auth.rs`、`mbti.rs` | struct/enum、借用、集合、数据库参数绑定 |
| 待填写 | 院校推荐算法与 MySQL 查询 | `recommendation.rs` | trait 思想、迭代器、所有权、数值计算、SQL 安全 |
| 待填写 | 专业知识图谱与文本分析 | `knowledge.rs` | 图遍历、HashMap/HashSet、生命周期与借用 |
| 待填写 | 风险模拟、模型对话和测试 | `portfolio.rs`、`llm.rs`、各模块测试 | 泛型思想、随机模拟、异步网络、单元测试 |

## 协作约定

1. 每个功能从独立分支开发，例如 `feature/mbti`、`feature/recommendation`。
2. commit 只包含一个明确目的，建议使用：
   - `feat: 实现 MBTI 计分`
   - `fix: 修正志愿概率边界`
   - `test: 增加知识图谱遍历测试`
   - `docs: 补充数据库部署说明`
3. 合并前至少由另一名成员检查，并运行 `scripts/check.ps1`。
4. 不提交 `.env`、数据库数据目录、`target/` 或任何真实 API 密钥。
5. 实验报告中分别说明接口设计、模块协作和每位成员解决的具体问题。

## 建议提交节奏

- 第 1 阶段：项目骨架、配置与错误类型。
- 第 2 阶段：认证、MBTI、推荐、知识图谱各自独立提交。
- 第 3 阶段：API 集成与前端联调。
- 第 4 阶段：测试、Clippy、文档和演示视频脚本。
